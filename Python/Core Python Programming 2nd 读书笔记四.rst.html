所有的Python对像都拥有三个特性：

  • 身份：可以使用 id() 内建函数来得到，可以被认为是该对象的内存地址
  • 类型：可以使用 type() 内建函数来得到，返回的类型也是一个对象
  • 值：对象表示的数据项

还有一部分我们了解很少的类型，可以称为是Python的内部类型。

  • 代码：代码对象是编译过的Python源代码片段，它是可执行对象。通过调用内建函数 compile() 可以得到代码对象。代码对象可以被 exec 命令或 eval() 内建函数来执行
  • 帧：帧对象包含Python解释器在运行时所需要知道的所有信息
  • 跟踪记录：当异常发生时，一个包含针对异常的栈跟踪信息的跟踪记录对象被创建
  • 切片： [::-1] 可以达到reverse的效果
  • 省略
  • Xrange： 有内建函数 xrange() 生成

不同于很多其它语言，Python中多个比较操作可以在同一行上进行，求值顺序为从左到右

>>> 4 < 3 < 5 != 2 < 7
False
>>> 2 < 5 < 10 != 7 > 3
True

运算符 is 和 is not 比较的是对象是否指向同一个地址

在做对象比较的过程中，有一点要引起注意，就是 整数对象和字符串对象是不可变对象，，所以Python 会很高效的缓存它们，目前对于Python2.5.2的整数的缓存范围是(-5, 256)

函数 功能
cmp(obj1, obj2) 比较 obj1 和 obj2, 根据比较结果返回整数 i: i < 0 if obj1 < obj2 i > 0 if obj1 > obj2 i == 0 if obj1 == obj2
repr(obj) 或 `obj` 返回一个对象的字符串表示
str(obj) 返回对象适合可读性好的字符串表示
type(obj) 得到一个对象的类型，并返回相应的type 对象

str、repr、`` 大对比：

repr 和 `` 是一回事，不过 repr 为内建函数， `` 为运算符，目前Python的社区已经开始不鼓励使用 `` 了。他们都是返回一个对象Python需要的字符串表示，也通常能用 eval() 内建函数重新得到该对象， newobj = eval(repr(oldobj))

str 则会返回一个对象的使人更易阅读的字符串表示，通常更适合print语句输出

注意： type()返回的是某个类型对象，且Python中一个类型只有唯一一个类型对象

所有在比较类型的过程中，比较对象身份比比较值性能更优，因为一个类型只有唯一一个类型对象，所以如果两个类型对象的身份是一样，即是同一类型，那么它们类型对象的值也必定相同。

>>> import types
>>> type(123) is types.IntType
True

注意： == 号操作符比较的是值而 is 语句比较的是身份

同时如果要查询多个类型，用 isinstance() 内建函数可以避免写出很多if-elif-else语句，从而提升性能。看个书中的例子：

#!/usr/bin/env python

def displayNumType(num):
    print num, 'is',
    if isinstance(num, (int, long, float, complex)):
        print 'a number of type:', type(num).__name__
    else:
        print 'not a number at all!!'

displayNumType(-69)
displayNumType(9999999999999999999999L)
displayNumType(98.6)
displayNumType(-5.2+1.9j)
displayNumType('xxx')

这里还有一点关于元类编程的知识点补充，详细的可以看本书的第十三章

新建一个类是要一个模板的，这个模板就是元类。对于

class A: pass

它使用的元类就是 types.ClassType ，字符串表示为<type 'classobj'>；而

class B(object): pass

由于没有在类里定义 __metaclass__ 属性，就会查看父类的 __metaclass__ ，父类是object，它的 __metaclass__ 为type， type(object) is types.TypeType

所以可以通过加入 __metaclass__ 属性来使传统类变为新样式的类

Python 2.2统一了类型和类，所有的内建类型现在也都是类。在这基础之上，原来的所谓内建转换函数象int()、type()、list()等等，现在都成了工厂函数。也就是说虽然他们看上去有点象函数， 实质上他们是类。当你调用它们时，实际上是生成了该类型的一个实例，就象工厂生产货物一样。

有三种不同的模型可以帮助我们对基本类型（也就是标准类型：基本内建数据对象原始类型）进行分类：

  • 存储模型：就是看看这种类型的对象能保存多少个对象，一个能保存单个字面对象的类型我们称它为原子或标量存储，那些可容纳多个对象的类型，我们称之为容器存储

    分类 Python

    类型

    标量/原子类型

    数值（所有的数值类型），字符串（全部是文字）

    容器类型

    列表、元组、字典

  • 更新模型：就是看看对象创建成功之后，它的值是否可以进行更新

    分类 Python

    类型

    可变类型

    列表， 字典

    不可变类型

    数字、字符串、元组

  • 访问模型：中共有三种访问方式：直接存取，顺序，和映射

    类型

    直接访问

    数字

    顺序访问

    字符串、列表、元组

    映射访问

    字典
