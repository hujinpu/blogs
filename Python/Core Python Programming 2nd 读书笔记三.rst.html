Python语句中有一些基本规则和特殊字符：

  • 井号(#)表示之后的字符为 Python 注释
  • 换行 \n 是标准的行分隔符（通常一个语句一行）
  • 反斜线 \ 继续上一行
  • 分号 ; 将两个语句连接在一行中
  • 冒号 : 将代码块的头和体分开
  • 语句（代码块）用缩进块的方式体现
  • 不同的缩进深度分隔不同的代码块
  • Python 文件以模块的形式组织

有两种例外情况一个语句不使用反斜线也可以跨行：

  • 在使用闭合操作符时，单一语句可以跨多行，例如：在含有小括号、中括号、花括号时可以多行书写
  • 三引号包括下的docstring字符串也可以跨行书写

建议缩进四个空格宽度，避免使用制表符

在Python语言中，对象是通过引用传递的。在赋值时，不管这个对象是新创建的，还是一个已经存在的，都是将该对象的引用（并不是值）赋值给变量。

Python中，赋值语句不会返回值，但链式赋值是被支持的。

>>> x = 1
>>> y = (x = x + 1)
  File "<stdin>", line 1
    y = (x = x + 1)
           ^
SyntaxError: invalid syntax
>>> y = x = x + 1
>>> x, y
(2, 2)
>>>

增量赋值：等号可以和一个算术运算符组合在一起，将计算结果重新赋值给左边的变量。 例如： x += 1

增量赋值相对普通赋值不仅仅是写法上的改变， 最有意义的变化是第一个对象仅被处理一次 。可变对象会被就地修改（不会产生拷贝的引用），不可变对象则和 x = x + 1 的结果一样（分配一个新对象）

在多元赋值中，建议左边也加上括号来增加可读性，如：

>>> (x, y, z) = (1, 2, "hello")
>>> (x, y, z)
(1, 2, 'hello')
>>>

标识符是电脑语言中允许作为名字的有效字符串集合，合法的标识符为：

  • 第一个字符必须是字母或下划线（_）
  • 剩下的字符可以是字母和数字或下划线
  • 大小写敏感

所有的关键字列表可以通过keyword的kwlist属性来获得，该模块还有一个 iskeyword(str) 函数用来判断str是否为关键字

除了关键字之外，Python还有可以在任何一级代码使用的“内建”的名字集合，这些名字可以由解释器设置或使用。虽然built-in不是关键字，但是应该把它当作“系统保留字”，不做他用。然而，有些情况要求覆盖（也就是：重定义，替换）它们。 Python不支持重载标识符，所以任何时刻都只有一个名字绑定。

built-in是__builtins__模块的成员，会在你的程序开始或在交互解释器中给出>>>提示之前，由解释器自动导入， dir(__builtins__) 就可以得到这些内建的“系统保留字”的列表

  • _xxx 不会从'from module import *'导入
  • __xxx__ 系统定义名字
  • __xxx 类中的私有变量名

所以，当变量是私有的时候，用_xxx 来表示变量是很好的习惯，其他情况应该避免使用下划线为一个变量名的开始

Python的代码风格指南可以见 PEP8

推荐一种模块的合理的布局：

  1. 起始行（Unix/Linux）
  2. 模块文档（docstring）：模块外可通过 module.__doc__ 访问这些内容
  3. 模块导入：每个模块仅导入一次（当前模块被加载时），函数内部的模块导入代码不会被执行，除非该函数正在执行
  4. 变量定义：除非必须，尽量使用局部变量代替全局变量，可使代码更易维护和提高性能并节省内存
  5. 类定义：当模块被导入时class语句会被执行, 类也就会被定义
  6. 函数定义：当模块被导入时 def 语句会被执行，函数也就都会定义好
  7. 主程序：无论这个模块是被别的模块导入还是作为脚本直接执行，都会执行这部分代码

贴一个完整的例子：

#/usr/bin/env python

"""this is a test module"""

import sys
import os

debug = True

class FooClass(object):
    """Foo class"""
    pass

def test():
    """test function"""
    foo = FooClass()

    if debug:
        print 'ran test()'

if __name__ == '__main__':
    test()

如果模块是被导入， __name__ 的值为模块名字，如果模块是被直接执行， __name__ 的值为 '__main__'

  • 变量无须事先声明
  • 变量无须指定类型
  • 程序员不用关心内存管理
  • 变量名会被“回收”
  • del 语句能够直接释放资源

Python语言中，对象的类型和内存占用都是运行时确定的。在创建－－也就是赋值时，解释器会根据语法和右侧的操作数来决定新对象的类型。在对象创建后，一个该对象的引用会被赋值给左侧的变量。

要保持追踪内存中的对象，Python使用了引用计数这一简单技术。一个内部跟踪变量，称为一个引用计数器。至于每个对象各有多少个引用，简称引用计数。对象被创建时， 就创建了一个引用计数，当这个对象不再需要时，也就是说，这个对象的引用计数变为0时，它被垃圾回收。

对象的引用计数在以下情况会增加：

  • 对象被创建 x = 3.14
  • 或另外的别名被创建 y = x
  • 或被作为参数传递给函数（新的本地引用） foobar(x)
  • 或成为容器对象的一个元素 myList = [123, x, 'xyz']

对象的引用计数在以下情况会减少:

  • 一个本地引用离开了其作用范围。比如foobar()函数结束时
  • 对象的别名被显式的销毁 del y # or del x
  • 对象的一个别名被赋值给其它的对象 x = 123
  • 对象被从一个窗口对象中移除 myList.remove(x)
  • 窗口对象本身被销毁 del myList # or goes out-of-scope
